Create a class called ReportService.
It should use a custom ThreadFactory and a fixed thread pool to run CompletableFuture tasks with named, controlled threads.
Create a ThreadFactory that names threads "report-worker-N" and sets them as daemon threads.
Use it to create a newFixedThreadPool(3).
Submit 6 report generation tasks using supplyAsync() passing the custom pool as the second argument, each sleeping a random time between 300ms and 800ms and returning "Report-N generated by <threadName>".
Chain thenAccept() to print each result.
Use allOf() to wait for all tasks and print total execution time at the end.
Shutdown the pool after everything finishes.
Observe that thread names in the output are always report-worker-N instead of the default ForkJoinPool-1-worker-N.

------------------------------------------------

PT-BR:
Crie uma classe chamada ReportService.
Ela deve usar uma ThreadFactory customizada e um pool de threads fixo para rodar tarefas de CompletableFuture com threads nomeadas e controladas.
Crie uma ThreadFactory que nomeia as threads "report-worker-N" e as define como daemon threads.
Use-a para criar um newFixedThreadPool(3).
Submeta 6 tarefas de geração de relatório usando supplyAsync() passando o pool customizado como segundo argumento, cada uma dormindo um tempo aleatório entre 300ms e 800ms e retornando "Report-N generated by <threadName>". Encadeie thenAccept() para imprimir cada resultado.
Use allOf() para esperar todas as tarefas e imprima o tempo total de execução ao final.
Faça o shutdown do pool depois que tudo terminar.
Observe que os nomes das threads no output são sempre report-worker-N em vez do padrão ForkJoinPool-1-worker-N.
