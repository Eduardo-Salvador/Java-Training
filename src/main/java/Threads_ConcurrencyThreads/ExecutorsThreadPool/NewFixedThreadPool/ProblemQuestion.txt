Create a class called ReportGenerator.
It should use a newFixedThreadPool(3) to generate reports concurrently, meaning only 3 reports can be generated at the same time regardless of how many are submitted.
Create a Callable<String> that simulates generating a report, it receives a report name, sleeps for 800ms to simulate work, and returns a result string.
For example: "report-4 generated by Thread-2".
Submit 9 reports to the pool and store each Future<String> in a list.
After submitting all tasks, iterate over the futures and print each result using get().
Shutdown the pool properly after all results are collected.
As a bonus, print a timestamp before and after each get() call to observe how the 9 tasks are processed in batches of 3 due to the fixed pool size.

--------------------------------------

PT-BR:
Crie uma classe chamada ReportGenerator.
Ela deve usar um newFixedThreadPool(3) para gerar relatórios de forma concorrente, ou seja, apenas 3 relatórios podem ser gerados ao mesmo tempo independente de quantos forem submetidos.
Crie um Callable<String> que simula a geração de um relatório, ele recebe o nome de um relatório, dorme por 800ms para simular trabalho, e retorna uma string de resultado.
Por exemplo: "report-4 generated by Thread-2".
Submeta 9 relatórios ao pool e armazene cada Future<String> em uma lista.
Após submeter todas as tarefas, itere sobre os futures e imprima cada resultado usando get().
Faça o shutdown do pool corretamente após coletar todos os resultados.
Como bonus, imprima um timestamp antes e depois de cada chamada get() para observar como as 9 tarefas são processadas em lotes de 3 devido ao tamanho fixo do pool.
